Self-Update: Reducing Error Messages

And what's next

February, 2017

The Main Problem

  • Self-Update is included in SLE 12 SP2 but disabled by default
  • In some scenarios is too intrusive
    • Error messages
    • Timeouts

The Worst Case

  • Installing behind a firewall which prevents access to your DNS
  • Packages are simply drop
  • You get 3 pop-ups:
    • Network error
    • Could not get updates repositories: falling back to control file
    • Could not access to fallback repository
  • And more than 30 seconds timeout
  • Check the video

Proposal

  • Report errors only when using custom URLs
    • Custom SMT
    • Custom update repository
  • Check the video
  • More information in the Self-Update Use Cases and Error Handling document

What's next?

Just some ideas

  • Reducing timeouts
    • Adjust the amount of times/retries
    • Should we fallback to the control file at all?
  • Define whether self-update is enabled/disabled through the control file
  • Consider adding information in case of a successful update
  • Include self-update options in the boot menu
